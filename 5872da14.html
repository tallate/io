¶背景

一场比较纯粹的Java面试，偏向于互联网技术，问了很多中间件的知识，但是我对这些东西都只停留在用过的层面上，因此很多都答不上来，这里记一下其中印象比较深的。

¶一面

面试时感觉状态不是特别好，很多平时总结过的知识点都没答上来，下面是我有印象的一些问题。

  1. JVM的内存结构（JMM）了解什么？
    JMM的主要目标是定义程序中各个变量（非线程似有）的访问规则，即在虚拟机中将变量存储到内存和从内存取出变量这样的底层细节。
    需要JMM的根本原因是线程间通信和线程同步需要多个线程共同操作一块共享内存。
    需要JMM的直接原因是指令重排序的存在，首先处理器本身会进行指令级并行的重排序和内存系统的重排序，Java编译器在将Java代码编译为字节码时也会在不改变单线程程序语义的前提下（代码中不包含synchronized关键字），重新安排语句的执行顺序。
    JMM主要解决：依赖关系、内存可见性问题、线程同步问题（根据我自己的理解，对这三个问题的理解是最重要的）。
    对于依赖关系，需要限制编译器和处理器对存在数据依赖关系的操作做重排序，因为这种重排序会改变程序执行结果，需要采用猜测（Speculation）执行的策略来克服控制相关性对并行度的影响。依赖关系和我们的开发没有什么关系，属于编译器和JVM包办的领域。
    对于内存可见性问题，主要解决方案是根据happens-before原则在生成的序列的适当位置插入内存屏障指令。平时比较常见对内存可见性的讨论，Java提供了一些同步机制来实现内存可见性：

    • volatile
      线程同步问题是比内存可见性更高级的一个问题，就是说解决线程同步问题隐含了解决内存可见性问题。解决线程同步问题的主要工具是锁，比如Java中就存在下面这些工具来实现锁的语义：
    • synchronized
      其他还有很多能讨论的东西，比如final这些关键词的实现原理、Java对象、类在内存中的布局等。

  2. 垃圾收集器有哪些？
    JDK1.4和1.5已经很少用了，基本上是Serial（Serial Old），JDK1.6是ParNew或者Parallel（Parallel Old），JDK1.7是Parallel、Parallel Old，1.8是Parallel、Parallel Old配合CMS，1.9+出现了G1。

    • Serial New收集器是针对新生代的收集器，采用的是复制算法
    • Parallel New（并行）收集器，新生代采用复制算法，老年代采用标记整理
    • Parallel Scavenge（并行）收集器，针对新生代，采用复制收集算法
    • Serial Old（串行）收集器，新生代采用复制，老年代采用标记整理
    • Parallel Old（并行）收集器，针对老年代，标记整理
    • CMS收集器，基于标记清理
    • G1收集器：整体上是基于标记 整理 ，局部采用复制

  3. Redis了解哪些？
    我：有5种主要数据类型：String、List、Hash、Set、ZSet。
    补充：Redis除了数据类型，可以讨论的还有持久化、线程模型、集群方案、缓存与数据库双写一致性、缓存雪崩、缓存击穿、缓存的并发写竞争问题、缓存降级等。

  4. Set和ZSet有什么区别？
    我：ZSet是有序的。
    补充：他们的根本区别当然是底层数据结构不同，等以后有时间再作总结。

  5. Redis更新是什么操作？返回值是什么？
    我：没答上来。
    面试官：set操作，返回是否成功。
    补充：set如果成功会返回"OK"，hset、incr、decr会返回更新结果，del、hdel会返回删除了的数据个数，get、hget、hgetAll返回查询到的数据，expire返回1表示设置成功0表示失败，ttl返回key的有效时间，-1表示永不过期-2表示已过期或已转移非负数n表示还有n秒过期。

  6. 用Redis怎么实现消息队列？
    我：…
    补充：Redis的List是使用双向链表实现的，实现消息队列只需要简单的两个指令lpush和rpop或者rpush和lpop。

  7. 用Redis怎么实现分布式锁？
    我：保存一个String类型的变量，如果这个变量已经有值了则加锁失败。
    面试官：如果网络超时了，你怎么知道加锁是不是成功了？
    我：那不能用String了，可以用一个Hash类型的变量LOCKPATH保存对每条数据的锁，保存结构如<LOCKPATH, <ID, 等待序列号>>，客户端要加锁时，先获取ID下的所有序列号，找到其中最大的值，再新建一个最大值+1的序列号保存到ID下。
    面试官：那多个客户端同时去获取序列号，怎么保证序列号不冲突？
    我：…
    面试官：Redis有一个setnx命令你了解吗？
    我：不了解。
    面试官：setnx(set if not exists)只有在值不存在的时候才会保存成功并返回保存后的值，否则返回0。如果超时的话只要判断返回值是否是目标值即可。

  8. RabbitMQ有哪些特性？
    我：镜像队列，可以提高可用性。
    补充：RabbitMQ可以讨论的内容也很多，运维方面需要知道怎么管理服务器、怎么看管日志、怎么进行消息追踪等，原理方面包括队列、镜像队列、通信、消息确认机制、网络分区、消息持久化等，没有一个主题是可以简单说清的，等以后有时间再作总结。

  9. 除了RabbitMQ，为什么不考虑使用别的消息队列？
    我：当时主要关注能否实现，没有去深究各个消息队列之间的区别。
    补充：消息队列还有很多产品，每个产品的特性还多少有点不同，等以后有时间再作总结。

  10. Nginx重载命令是什么？

  12. 数据库隔离级别有哪些？
    我：…
    补充：数据库隔离级别分为4种：读未提交(Read uncommited)、读已提交(Read committed)、可重复读(Repeated read)、序列化(Serializable)。这些隔离级别是如何实现的？为什么这么实现？等以后有时间再作总结。

  13. 了解数据库索引吗？
    我：索引的底层实现是B+树。
    补充：索引主要分为聚集索引和非聚集索引，聚集索引将key和数据放在一起，非聚集索引只保留key和数据的地址（只是比较粗略的考量，等以后对数据库有了更深入的理解再作总结）。

  14. 索引为什么不用B树？
    B+树相对B树来说最大的优势是能更好地支持范围查询，B树因为内部节点也能存储数据，如果需要查询某个范围内的值可能需要中序回溯。

  15. 索引为什么不用红黑树？
    B+树将相近的key放在相邻的地方，根据“局部性原理”，有望减少IO数，提高整体吞吐量（当然实际怎么样得看数据的特征）。

  16. 索引为什么不用哈希表？
    B+树是有序的，针对order by、group by等操作会有更好的效率。

¶笔试

笔试题大部分考一些Java、Spring、分布式系统基础等，最后一题设计一个系统比较灵活。因为面试过程太长了，我现在记不得全部的题目了，将其中印象比较深刻的记录如下。

  1. Java字符串常量和实例化的字符串对象之间的区别（"123"和new String(“123”)）？
    常量分配在虚拟机的方法区内（1.8及以后的JDK版本用元空间取代），而new操作会将对象分配在堆中。
  2. 使用abstract类的目的是？
    实现公共方法、且不允许实例化。
  3. 子类实现父类中方法是为了？
    Hiding，重写后不能直接调用父类方法，除非在子类中使用super显式调用。
  4. Spring scope有哪些？
    singleton:单例
    prototype:多例
    request:一次HTTP request产生一个新的bean
    session:一次HTTP session产生一个新的bean
    global session:这个只在portal应用中有效，给每一个global http session新建一个bean实例。
  5. Spring注入方法有哪些？
    byName:通过注册到容器中的Bean名字来获取其实例。
    byType:通过Bean的类型来获取其实例。
  6. Spring中singleton是并发安全的吗？
    不是。之前尝试过多次请求一个Controller来对一个类变量执行++操作。
  7. SpringData、Hibernate、MyBatis之间的区别？
    Hibernate提供了基础的查询方法，但也可以使用Hql来自定义复杂查询。
    MyBatis创建了POJO和SQL之间的映射。
    Spring Data定义了一种命名规范，比如findByUser，会被从右向左解析生成sql语句，当然也可以使用注解@Query来绑定sql语句。
  8. 服务无状态的含义是什么？
    服务内没有保留任何状态信息。

¶第二次笔试

之后技术老大可能是觉得我的代码能力值得怀疑，于是又约时间进行了一次笔试，这一次考的是算法题。

¶一、今有有向图，如何判断两点之间是否互通？

计算图之间距离的算法常见的比如：

  1. Dijkstra：计算两点之间的最短路径；
    • 负权边的存在：每轮执行循环体的过程中需要更新当前节点和其相邻节点之间的最短距离，如果图中存在负边，得到的结果是意义不明的（首先我们必须要定义负权的含义），，甚至于如果存在一个总长为负数的圈，则算法会陷入死循环，如果给每条边都加上一个值可以确保算法能执行完，但同样不能保证找到的路径在原图中是最短的，这里有解释。
    • 使用优先队列优化：每轮贪心选择当前已得到从初始节点发出最短路径的节点，可以使用一个优先队列来加快这个。
  2. Floyd：计算任意两点之间的最短路径，这题只求可达性，所以可以忽略距离直接用1表示可达。
    • 空间浪费：因为Floyd的算法实现基本都是基于二维数组的，如果图是稀疏的，就会导致严重的空间浪费。
  3. Bellman-Ford
  4. SPFA

本题需要判断两点之间是否互通，所以其实没有必要使用上面的算法，直接用DFS（BFS也是类似的）就可以达到目的。
首先要解决的是图的表示问题，图主要有3种表示方法：

  1. 邻接矩阵：使用一个二维数组来表示所有顶点之间的连接关系；
  2. 邻接表：每个顶点记录其所有相邻的边，这种方法需要注意因为链表随机访问的效率是O(n)的，所以对算法的实现是存在限制的；
  3. 邻接多重表：这个比较少见，edge变成了5元组<mark, vertex1, vertex2, path1, path2>，vertex变成了2元组<data, firstout>，在某些算法中可以减少操作步骤增加效率。

本题对图结构没有规定，为了方便可以直接用二维数组来表示。

但是为了提高效率也可以适当地对邻接表做一些魔改，注意Edge结构中还多了一个dest域来保存目标节点在Graph.nodeTable中的下标。

在编码之前还必须考虑一些实现细节：

  • 环的存在：需要使用一个visited数组来记录已经访问过的顶点。
  • 迭代而不是递归：递归相对迭代来说是更高效的方法，使用Stack可以模拟DFS的执行过程。

¶二、一道三维装箱问题

原题可以在牛客上找到，但是暂时没有找到靠谱的题解。
初看这道题会联想到背包问题，因为都是将东西装到包里，但是稍微想一下就会发现差异还是很大的：1.传统背包问题里的背包是有限个数的，而该题没有；2.传统背包问题关注的是物品的质量，而该题装的是物品的形状。
如果忽略条件2，那问题就退化成一个多背包问题，使用贪心+整数规划基本可以肯定会得到一个局部最优解而不是全局的，所以应该考虑使用动态规划+整数规划的解法。
三维条件下情况比较复杂，我在实际解答时苦恼的也是找不到解集树中的重合部分，花了比较长时间才给出了一个勉强的答案，现在有时间了，就把主要思路描述如下。
限制：1.价值;2.形状。
无解：1.某商品价值超出2000;2.某商品无法放入盒子中。

算法1(回溯剪枝)：遍历解集树，找到所有解，回溯——每次要么从现有盒子中找到一个装入当前商品、要么
算法2(动态规划)：这题状态非常复杂，如果要使用动态规划来实现，就必须定义一种状态的映射机制，将boxes链表映射到已计算出的结果上。

对于优化问题，我们一般会先想到回溯、贪心、动态规划、分治等，然后将解题的过程形象地理解为遍历一棵解集树，树中的每一节点表示某一状态，回溯表示遍历完一棵子树后恢复到上一节点的状态，贪心表示搜索目标在某一条确定的路径上（树退化成链表），动态规划表示树中某些节点是重复的、可以缓存起来减少对某些路径的重复搜索（可以从树的结构导出一个递归式），分治从两棵子树查找两个子问题的解、将这两个子问题合并来得到最终的解。
数学中称这种从“解集树中寻找解的问题”称为最优化问题，已经有很多书去介绍有哪几类问题、对这些问题有哪些算法。根据变量性质可以分为连续优化问题和离散（组合）优化问题，连续优化问题中有一类比较有名的凸优化问题，因为时间有限我没法把具体的算法都总结一遍，其中涉及到的概念比较多，暂时先只对网上的资料进行简单的总结。

¶主要涉及内容
