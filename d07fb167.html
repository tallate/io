OpenSSH使用入门

发表于 2018-12-05 | 浏览 次
字数统计: 1.3k 字 | 阅读时长 ≈ 5 分钟

现在的常见*nix、Mac系统中默认都会装有OpenSSH[1]拿来与远程服务器进行安全连接、传输文件、执行命令等任务。
在1990s年代，SSH取代了Unix和Linux很多老的命令和协议，包括telnet、rlogin和rsh。
SSH会运行一个TCP/IP服务器监听22端口，和ftp和telnet协议相邻。

¶四、host key 和 identity key

在配置免密登录服务器的时候，我们经常会碰到两个概念：host key和identity key。它们看起来似乎是类似的概念，但是其实是不同的概念：

  1. hk是为主机授权，公钥是保存到用户机器上的；
    ik是为用户授权，公钥是保存到服务器上的。
  2. hk是为了避免用户受到**“中间人攻击”**5；
    而ik是为了避免服务器被未知用户登录操作。
  3. hk在第一次登录服务器的时候会提示Are you sure you want to continue connecting (yes/no)?，如果输入yes会新建一个host key到用户的.ssh/known_hosts文件内，并在每次登录服务器的时候验证；
    ik是由用户自己设置的，同样也是在每次登录服务器的时候验证；
  4. hk的私钥一般保存到/etc/ssh目录下的ssh_host_<rsa/dsa/ecdsa/ed25519>_key文件内，公钥需要保存到/etc/ssh/known_hosts和每个用户的.ssh/known_hosts文件内；
    平时私钥一般指的是ik，可以使用ssh-keygen工具来生成，这个工具会生成一个私钥和一个公钥，然后可以使用ssh-copy-id命令将公钥作为授权密钥安装到服务器上，它会将公钥添加到服务器上的authorized_keys文件内。通常，每个SSH实现都有自己的工具来生成和安装密钥。

¶五、配置端口转发

命令行选项能用于设置端口转发。

  • Local forwarding：建立一个从本地端口到某个服务器IP:port的隧道（tunnel）；
  • Remote forwarding：建立一个从服务器某端口到给定客户机器IP:port的隧道。

OpenSSH同时支持对来自某个隧道硬件的Unix domain sockets和IP packets进行转发，从而建立VPN（Virtual Private Network）。

¶六、SSH命令行选项

一些最重要的OpenSSH客户端的命令行参数如下：

  • -1：使用版本1的协议；
  • -2：使用版本2的协议；
  • -4：只使用IPv4地址；
  • -6：只使用IPv6地址；
  • -A：启用认证代理连接的转发；
  • -a：禁用认证代理连接的转发；
  • -C：使用数据压缩；
  • -c：cipher_spec 选择对会话进行加密的密文规则；
  • -D：[bind_address:]port
  • -E：log_file 添加debug日志到log_file而不是标准error。
  • -F：configfile 指定每个用户的配置文件，默认是~/.ssh/config。
  • -g：允许远程主机连接到本地转发端口。
  • -i：identity_file 私钥文件，需要和公钥对应。
  • -J：[user@]host[:port] 先连接到pjump主机(/iam/jump-host)，然后再通过TCP转发连接到目标服务器。
  • -l：login_name 远程主机的用户名。
  • -p：port 目标远程主机端口号。
  • -q：quiet模式，不输出到终端。
  • -V：显示版本号。
  • -v：verbose模式。
  • -X：启用X11转发。

¶七、其他相关命令

  • ssh-keygen：为公钥认证生成密钥对。
  • ssh-copy-id：在服务器上配置一个授权公钥。
  • ssh-agent：指定持有私钥的代理服务器，用于一次性登录。
  • ssh-add：添加一个私钥到代理服务器。
  • scp：提供一种类RCP的文件传输客户端命令界面。
  • sftp：类FTP的命令界面。
  • sshd：OpenSSH服务器。
