这一篇主要是描述测试方面的记录，主要是性能方面的，因为这方面在实践中总是比较敏感。

¶性能指标

性能测试中使用到了一些指标，其中的一些很容易被混淆的，比如“响应时间”和“执行时间”，所以在测试之前，我们先来汇总一下：

¶测试相关命令

  • redis-benchmark
    redis-benchmark 是 Redis 官方提供的 Redis 服务器性能基准测试工具：
    -t 选择你想测试的命令，比如 redis-benchmark -t set
    -p 指定 port redis-benchmark -p 6379
    -l 一直循环
    -c 指定客户端数量
    -n 指定 request 数量
    -q Quiet，不显示额外信息（多少时间内完成了多少条之类的），只显示 query/sec 的值

¶设置对比组

正像我们学物理、生物做实验的那样，所有实验都必须要有对比，没有参照系讲运动速度就是耍流氓，所以测试前先考虑一下不使用这个框架的效率。

框架的执行效率很多时候是比不上原生开发的。框架很大程度上的目的是为了减少重复代码、提升开发效率，将常用代码封装避免重复造轮子（更别提造出有问题的轮子成了祸害）。编写框架的过程就是抽象出一个通用模型的过程，但是框架为了通用有时候又会牺牲执行效率，比如 Spring 的效率就经常被人诟病，但它却仍被大多公司作为后台业务框架的基础。
