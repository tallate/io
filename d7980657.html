2018下旬跳槽经历小结（Updating）

发表于 2018-11-02 | 浏览 次
字数统计: 3.8k 字 | 阅读时长 ≈ 12 分钟

前一段时间一边在办离职找工作的事情，另一边又要应付公司内的任务，忙得脑壳凉。现在做一个复盘，希望能总结经验、得到一些启示。
（与其说是总结，不如说是一些抱怨，因为身边的事情没有按照自己的想法运行，很多时候推脱说自己很年轻，其实是愚蠢，还掉到不少坑里。没有可靠的信息来源，只能自己走一步算一步。）

¶经过

  1. 谈面试经过
    首先说一下经过，总得来说，我的大部分机会都是通过Boss直聘、猎聘这两个平台自动找上门来的，我没有太多大公司人脉，但多少也投了5家稍大公司，但是其中三家都因为年限问题卡了，一家拿到了offer但是工作内容不大让人满意，另一家技术不过关二面被刷。所以最终目标还是定位于不大有名但技术不差的中小公司。
    让我印象最深刻的是第一家，是一家处于C轮的o2o公司，正在面临一个从o2o业务向人工智能化转型的阶段，面试官显然是比较会的，会从简单的问题一直剖到OS、计算机组成原理的层次，相对深入的问题我基本都答不上来，但是面完后面试官对我相对来说比较满意（虽然水平差点但是有工程师的心态），还承诺了比较理想的工资。但是后来因为我吃着碗里的看着锅里的、还去参加第二家的面试（第二家我经历了9面，横跨3个星期，最后被总裁一票否决了），对方怒了，就没有然后了。
    其实最理想的也是第一家，技术厉害又没有架子的Boss，慷慨的工资，扁平化的管理，稳定的资金流，给力的产品，满足我的一切需求，可惜Boss找上我太早了，如果我能先面个两三家、心里有数了，说不定结果会有些不同？
  2. 谈目标
    俗话说不忘初心方得始终，其实我的初心非常简单，我希望在职业生涯至少头三年要在技术层面上打下比较好的基础，但是在原东家做的都是业务方面的工作、得不到多少锻炼（不是瞎抱怨，在这里参与的项目在面试中都没有被怎么问到），于是在满一年后（算上实习时间）就心急火燎地跳了。
  3. 今后计划？
    就像原东家Boss警告我的那样，频繁的跳槽会成为职业生涯的污点、让用人单位产生排斥心理。因此我初步计划在新东家待至少3年，但是没法确定，因为心态总会受到环境影响。

¶不能完全相信任何一个人

人与人之间基本是靠利益驱使联系在一起的，只要自己失去了价值，那么人缘自然就失去了。但是反过来说，只要能看清人与人之间的利益链，就多少可以做到游刃有余了。
年轻时可以依靠身体来拿命换钱，青春是价值的一种，但是到了30多岁后就必须找其他出路了，因为各种职业病来了后，长期加班肯定是吃不消的了。我关注的很多学习APP，比如极客时间，上边会有很多资深的专家分享自己的经验，相当于将自己的知识变现，这有很多好处，比如赚外快、打出名声、复盘自己的所见所学、开拓职业发展思路。
在同事间交流离职的想法时，大家会认为我有进入大公司（BAT）的潜力而对我友善，但是发现我并没有这个能力，所以就没有必要忍受我糟糕的性格了。
其他的，比如亲情、学历、知识，都是自己的财富，都是可以好好利用的。
我知道上面的想法有点过激，但是我遇到的可以单纯当朋友的人实在太少了，也是当初因为怠堕而没有好好活用身边的资源，没有适时提升自己的价值，从而导致了现在的境地。

¶谨言慎行

在情绪高涨的情况下我偶尔会口不择言，有几次也因此得罪了人，所以，控制情绪仍然是我的必修课。

¶谈梦想是不靠谱的

梦想、未来这些抽象的东西是很轻巧的，很容易被现实的东西搏倒，到最后，发现还是谈钱谈工资好使，因为工资。可以量化自己的价值、可以看到自己受重视的程度，在这个前提条件下，才能更好地实现自己的人生价值，以至于谈梦想、未来，不是吗？

¶做任何事都必须要有计划

很多时候，我一旦要一次做多件事时就会焦头烂额，结果每件事都做不好。实际上是缺少计划，缺少对时间的管理。
对于短期任务——比如有好多个Bug等着要改，人是单线程的生物，如果要实现多线程，必须要设定好一个合适的调度机制，比如番茄工作法。
对于长期任务，就真的要看个人能力了，就像有些能写短篇小说的人并没有办法写出合格的长篇小说，它们其实是不同的两件事，执行长期任务的主要过程可以大致划分为：确定目标 -> 获取信息 -> 制订计划（阶段任务） -> 执行并不断修正计划 -> 复盘。在这次离职经历中，我没考虑清楚公司内项目的进度，一边计划离职（复习、接各种电话、面试等），一边还要支持当前公司内正在推进的项目，导致两边都不讨好，最后得到了一个差不多的结果，也是非常惭愧。

NCC是一个比当初我实习期间参与开发的司库云还要坑的项目，它没有任何吸引人的技术点，项目组里所有人都是受工资驱使干活的，所有看起来不在乎的人，其实要么是因为能力不足找不到下家，要么是已经在计划着离职了。项目启动阶段部门扩充编制招了快10个人，有一个人就是给我做交接的（我作为资源释放出去后也方便给其他小组改Bug），不论如何，有责任心的人不会撂挑子，至少要把项目的坑填完，对吧？

¶套路

现在不管是HR面还是技术面都会涉及到一些“模板题”，如果准备得好，就属于送分题，如果不重视不准备，也可能变成“送命题”。
比如，谈及“你对该公司的了解”，如果面试官发现你对他们要做的事情一点都不了解，甚至对o2o、c2c这些概念都不了解，那么对方可能会想：是不是根本就不尊重这次面试？或者根本就不想过来？
又如，你离职的原因？虽然无非就是内容无聊、加班多、上级不给里等，但不能抱怨，而是要说得比较深思熟虑，诉说自己的长期计划，让这次离职显得更有道理，就像打仗一样，战争是劳民伤财的，没有正经理由的战争会招人反感，进而失去民心。

¶知己知彼

面试说白了还是和人打交道，虽然程序员职位对情商的要求可能会稍微低一些，但是说错话触犯面试官、甚至因此把机会白白丢了就非常可惜了。
一方面，需要理解对方此次招人的目的，对方是需要熟练工还是潜力股，需要关键人才还是打杂的，大公司和小公司（是否上市、是否有名）不同，o2o公司和2B公司不同，融资规模不同，期望不同，结果也会有所差异，可以考虑问一下预留的位置数、要做什么事情、对方对职位的期望、对自己的评价等。
校招需要更多的是基础知识、潜力，那么一般就是把一些常见问题熟读背诵、准备一些竞赛的经验，还有积累一些有意思的项目，在面试的时候不至于尴尬无话可聊。社招就更多的会偏重于经验了，在参与过的项目中是不是处于重要的地位、对实际项目的理解等就比较重要，学历只是加分项。
另一方面，需要认清自己的定位。牛的人选公司，菜的人等公司选，这是社会选择的结果，当然不论如何，谦虚一些总不会错。

¶对就是对、错就是错

这是一位Boss给我的一句警告（当然没有这么严重，但是我更倾向于重视它），我有时会口不择言——特别是在睡眠不足的情况下，对一些问题，如果我以前接触过、而现在明显忘了，我更倾向于表现出：“我很懂，只是在细节方面有点瑕疵”，严格点说就是胡说八道。如果是对方具有工程师的品格，对方会直接指出我的错误，如果对方比较社会，就会直接换个话题，但避不了在心中给我打上低分。所以，总而言之，谦虚一点，有几斤几两就倒出几斤几两，其他的说自己忘了就OK了。

¶留一手，留一个心眼

可能是我太耿直了，在offer的谈判上，一向是去就是去、不去就是不去，不太会打太极。其实，为了保证自身利益的最大化，offer必然是越多越好，因此，就算对方催自己去作任何决定，仅作口头上的保证、留一手去其他单位面试，才是最佳的策略。
另一方面，作为打工仔，一定要留一个心眼，毕竟offer的法律效力并没有那么高，口头offer更是一张空头支票，任何口头保证都是不靠谱的。
思来想去还是觉得这条不恰当，我实在不大喜欢撒谎，也不喜欢其他人对我撒谎。

¶讲话技巧

在用友待的时间中我的直属老板是东北人，这个一聊就知道，一方面当然是口音，一方面我也常常感觉聊天的节奏会被带着跑。
前不久在面试一家互联网医疗相关企业时，HRD（人力资源总监）面时面试官是东北人，说话就非常的有技巧，会把握节奏，首先会用一两个和地域相关的简单话题来拉近两人的距离、炒热气氛，然后对一些疑惑的点剖根问底，如果我没有给出一个让她满意的回答就会一直问下去，在真实答案不方便说的情况下，可以给出一个稍微不那么让人难受的答案。比如如果早上没有精神，真实原因是看电视剧看得太晚了，但是这个显然不方便说，那么就可以拟说朋友失恋安慰他聊得太晚了之类的。

¶你还有什么问题？

这个阶段一方面可以更多地了解公司的现状，另一方面如果提问出色也可以给面试官留下一个更好的印象，毕竟你对我们公司一点都不了解还来面试做什么？
一些人给了我建议，如果非要以小公司作为目标，一定要对公司背景调查完善，大概要从人员结构、发展、工作内容、待遇这些方面进行提问，如：

  1. 团队规模多大？其中多少属于技术团队？
  2. 团队成员背景如何？CTO什么背景？
  3. 网站并发量 / 并发用户量 / 吞吐量 / QPS / TPS怎么样？
  4. 主营业务？有没有发展瓶颈？有哪些竞品？地位能排到top几？
  5. 进入公司后主要负责哪些方面的职责？会有哪些挑战？对我有什么要求？
  6. 有提供成长规划吗？1年后能到什么程度？3年后？

不断的提问可能会令对方感到不舒服，特别是一些尖锐的问题，所以，靠谱不靠谱也可以从一些细节方面看出来，比如：

  1. 面试轮数，一般3到4轮的情况下是靠谱的，4轮以上的属于高级工程师到架构师的层次、可能关乎某个项目的生死存亡了。
  2. package高低，可以表示对方的“诚意”，如果是技术老大或其他技术负责人给出的，还可以表示对方对自己的“期待”。

¶技术人心应该抱持怎样的心态？

社招相对校招来说会更看重候选人对所研究技术钻研的深度，从平时做的事切入，不断深入地挖掘内容。其实我很早就在准备，但是因为根底太浅，对很多连环提问仍然不能完整地接下来。我发现其中几场靠谱的面试中的提问都有这么一个规律：

  • 所有问题最终都可以归结到数学、数据结构和算法、操作系统、计算机组成原理、计算机网络、数据库、编译原理、图形学。

为什么会这么提问呢？我觉得更多的是看候选人的进取心和好奇心，这两个特质是面向未来的能力。当实现一个功能时，不妨问自己三个问题：

  • 为什么这么做？
  • 这样做背后有什么问题和思考？
  • 你的业务价值支撑点是什么？

反思自己，在实习期间也做过不少的内容，但是几乎都是纯业务的功能，没什么技术含量，要么是大家抄来抄去，要么是连需求自己都搞不明白，在好几场面试中因为无内容可聊还被面试官鄙视过，如果可以的话接下来还是希望能在技术方面。

¶技术人的三个品质

在面试一家的时候，面试官还问到技术人的三个品质是什么，当初我的回答是有责任心、刨根问底，但是没答上来第三个，现在想想应该再加上一个面向开放，能接受新事物。
为什么我认为是这三个品质呢？
