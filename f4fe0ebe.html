背景

这是今年的第一场面试，公司位置有点偏僻，网上流传了挺多差评，本来其实没抱什么希望，但是面试下来感觉很有意思，出的面试题不按套路，赶紧记一下。

一面

一面是比较常规的Java面试，先做了一张卷子，然后面试官根据卷子的内容问一些问题，但不瘟不火，没有太抠细节（可能是考虑到了我今年刚毕业）。下面是其中的一些问题。

  1. 接口能加什么修饰符？
    A. private
    B. protected
    C. final
    D. abstract
    接口本身就是抽象的，加不加abstract都一样。private和protected违反了接口需要开放的原则，如果加了final接口，接口不能被实现了（final的语义），那接口几乎也没意义了，总之这些修饰符都是不合法的。

  2. 给出下面代码的输出

    字符串字面量会被保存到方法区（常量池），而直接调用构造器构建的字符串对象则会分配到堆上，主要问题出在d和e上，d是两个字符串的连接，编译器会暗中使用StringBuilder完成这个任务，e中两个字符串字面量的拼接会被优化为一整个字符串（常量优化）。因此，其输出应该是下面这样的：
    true
    false
    false
    true

  3. 对象初始化顺序

  4. Object类中有哪些方法？写出它们的作用。
    clone()：复制对象。
    getClass()：获取对象所属的Class。
    hashCode()：计算hash值，默认为地址。
    equals()：比较对象是否相等，默认为比较地址。
    toString()：默认情况下结果为”类名@地址”。
    wait()：挂起当前线程，如果没有占有该对象上的监视器锁则抛出异常。
    wait(long timeout)：如果没有在timeout时间内被其他线程调用该共享对象的notify()或notifyAll()方法唤醒，则该函数仍会因为超时而返回。
    wait(long timeout, int nanos)：如果nanos>0则让timeout参数递增1。
    notify()：唤醒一个在该共享变量上调用wait方法挂起的线程，如果有多个线程则唤醒的目标是随机的。如果没有占有该对象上的监视器锁则抛出异常。
    notifyAll()：唤醒占有该对象监视器锁的所有线程。
    还有一些不常见的：
    finalize()：在对象被回收时调用。
    registerNatives()：用于注册native方法。

  5. 写一个单例模式
    当时直接用枚举enum写了个单例。

    最常见的单例模式实现方式包括懒汉式和饿汉式，懒汉式的实现还需要注意线程安全性，需要考虑使用synchronized还是双检锁；静态内部类方式可以实现延迟初始化但是实例域必须是静态的；枚举类方式实现起来比较简单，具有线程安全性，自动支持序列化（可以看反编译字节码后的代码），但是没有延迟初始化的特性。

  6. 写一条Linux指令杀死所有运行中的Java进程。
    瞎写了一个jps | kill，当然肯定不对。
    上网查到ps -ef | grep java | grep -v grep | awk '{print $2}' | xargs -p kill -9
    上面这条命令可以简化为jps -q | xargs -p kill -9

  7. Linux指令查出文件名以.conf结尾、且内容含有8080的文件。

  8. 写Sql
    三张表关联、排序、过滤，需要注意表之间的关系。

  9. 使用前序、中序遍历序列画二叉树。
    LeetCode105
  10. 有一个100级的楼梯，一次可以爬1步、2步或3步，求爬到100级有多少种爬法。
    简单的DP问题。
  11. JVM垃圾回收过程。
    GC分为检测和清理两个过程，检测过程比较有代表性的算法是可达性分析算法，清理过程有代表的是分代回收算法。
    可达性分析算法会先对所有可以从Roots到达的节点作标记，然后回收掉所有没有标记的对象。
    分代回收算法会将堆空间分成年轻代和老年代。新对象会被放到新生代的Eden区，当达到回收条件时，将有引用的放到新生代的Survivor区，

二面提升了难度，而且面试官偏学院派，喜欢从底层角度思考问题，提的一些脑洞清奇的问题我几乎都没有答上来。

  1. （简历中的NCC项目）SOA架构是什么模式？
    代理模式。面试官画了个ESB的示意图（吓了我一跳，他怎么都知道？）。

    微服务和SOA的区别
    微服务可以看作是轻量级的SOA，微服务中，SOA强调使用一个ESB企业服务总线来整合所有接口，在网络拓扑中，ESB作为一个中介者来转发所有请求。
    在NCC这个项目中，NC-Cloud中间件可以看作一个ESB，它实现了对后台NC服务、附件服务等的一个反向代理，可以形成类似如下的架构图。

    因此我觉得SOA当作中介模式更说得过去，但是当时没有反应过来。
    但是NCC是一个有点挂羊头卖狗肉的项目，它不会将后台服务分开来部署，甚至NCCloud中间件和NC服务器也是部署在一起的（真的是在一个文件夹下面的）。NCC中间件访问后台需要先获取后台接口的一个动态代理（JDK Proxy），如果配置文件中配置了远程服务器的地址，则它会将本地方法调用转换为远程调用（HTTP）。

    在数据库中，锁又叫封锁（Blockade），封锁的主要用处是同步多个事务对数据对象的修改，。

  3. 乐观锁是什么？
    项目中使用到的版本号和时间戳都属于乐观锁。
    乐观锁是一种思想，其核心是只在更新的时候对数据是否产生并发冲突进行检测，如果发现并发冲突了，则返回错误信息。
    CAS是实现乐观锁的一种技术，因为太常用所以很多时候CAS也指代乐观锁。CAS操作中只有一个线程可以更新变量的值，而其他线程都失败，失败的线程并不会被挂起，而是被告知这次竞争中失败，并可以再次尝试。Java中的JUC包是建立在CAS之上的。
