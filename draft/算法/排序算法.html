排序算法

排序算法概述

¶应用

  1. 数据库 select 查询的 order by 子句是排序；
  2. 许多涉及到列表展示的需求都包含排序；

¶排序算法的分类

排序的本质是消除数组中的所有逆序对，根据实现的方式的不同，大致可以分为：

  1. 基于比较的排序，大部分排序都是比较然后交换两个元素的位置，最后得到一个有序的数组。
  3. 思路清奇的算法，比如猴子排序，这些算法一般是开玩笑时用，基本没有实用性。

¶排序算法的指标

  1. 算法的执行效率。
  2. 排序的稳定性。
    比如我们按身高对一队人排序，身高相同的情况下，如果排完后他们的位置可能和排序前不一致，则这个算法是不稳定的，否则就是稳定的。
    对一些简单的值类型进行排序时对稳定性往往要求不那么高，但是如果是引用类型的，比如上述的对人的排序，稳定性还是比较重要的，这个要根据实际需要来讲。
  3. 排序元素的个数。
  4. 排序关键码的类型。
  5. 递归调用的开销。
    递归涉及到函数调用的栈帧维护，如果递归深度比较高会比较耗费性能，甚至导致栈溢出。

¶Java 中的 sort

Java 中 sort 接口一共结合了：插入排序，快速排序，和归并排序

  1. 如果是基本类型且元素个数小于 7，使用插入排序；
  2. 如果是基本类型且元素个数大于等于 7，使用快速排序；
    如何选取 pivot 是 Java 考虑的另一个主要因素，当数组大小 7 < size <= 40 时，取首、中、末 三个元素中间大小的元素作为划分元；当数组大小 size> 40 时 ，从待排数组中较均匀的选择 9 个元素，选出一个伪中数做为 pivot。

为什么这么选 pivot？我觉得是经验结论。

  1. 如果是引用类型，使用归并排序。

QA1：为什么元素数量少时使用插入排序，而多时使用快速排序？
数量少时插入排序不比快速排序慢，而且因为没有递归，所以开销更小；
QA2：为什么引用类型使用归并排序？
因为引用类型对稳定性要求较高，快速排序不稳定，而归并排序是稳定的。
